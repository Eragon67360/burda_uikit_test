Welcome to the FOCUS Abo-Shop UI Kit. This guide will help you integrate our components into your project.

## Installation

First, install the required dependencies:

```bash
npm install tailwindcss postcss autoprefixer
# or
yarn add tailwindcss postcss autoprefixer
```

## Using Components

Our components are framework-agnostic and can be integrated into any JavaScript project. Here's how to use them:

### Basic Usage

1. **Copy the Component Code**: Use the "Show Code" button in any component's story to copy the TypeScript implementation.

2. **Create a Wrapper**: Depending on your framework, create an appropriate wrapper:

#### For React:

```tsx
import { useEffect, useRef } from 'react';
import { createImage, ImageArgs } from './Image';

const Image = (props: ImageArgs) => {
  const containerRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    if (containerRef.current) {
      const imageElement = createImage(props);
      containerRef.current.innerHTML = '';
      containerRef.current.appendChild(imageElement);
    }
  }, [props]);

  return <div ref={containerRef} />;
};

export default Image;
```

#### For Vue:

```tsx
<template>
  <div ref="container"></div>
</template>

<script setup lang="ts">
import { onMounted, ref, watch } from "vue";
import { createImage, ImageArgs } from "./Image";

const props = defineProps<ImageArgs>();
const container = ref<HTMLDivElement>();

const renderImage = () => {
  if (container.value) {
    const imageElement = createImage(props);
    container.value.innerHTML = "";
    container.value.appendChild(imageElement);
  }
};

onMounted(renderImage);
watch(props, renderImage);
</script>
```

### Example Implementation

```tsx
// Using the Image component
const MyPage = () => {
  return <Image src="/path/to/image.jpg" ratio="16:9" alt="Description" />;
};
```

## Best Practices

1. **TypeScript**: Always use the provided TypeScript types for proper type checking:

```ts
import type { ImageArgs } from './Image';
```

2. **CSS Classes**: Our components use TailwindCSS classes. Ensure your build process includes these classes in your final CSS bundle.

3. **Accessibility**: Maintain WCAG compliance by providing proper alt texts and ARIA labels when extending components.

4. **Performance**: Components are lightweight by default. When wrapping them, avoid unnecessary re-renders.
