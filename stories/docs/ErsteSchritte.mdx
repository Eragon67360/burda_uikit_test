Willkommen beim FOCUS Abo-Shop UI Kit. Diese Anleitung hilft Ihnen bei der Integration unserer Komponenten in Ihr Projekt.

## Installation

Installieren Sie zunächst die erforderlichen Abhängigkeiten:

```bash
npm install tailwindcss postcss autoprefixer
# oder
yarn add tailwindcss postcss autoprefixer
```

## Verwendung der Komponenten

Unsere Komponenten sind Framework-unabhängig und können in jedes JavaScript-Projekt integriert werden. Hier erfahren Sie, wie Sie sie verwenden:

### Grundlegende Verwendung

1. **Komponenten-Code kopieren**: Verwenden Sie den "Show Code"-Button in der Story einer Komponente, um die TypeScript-Implementierung zu kopieren.

2. **Wrapper erstellen**: Erstellen Sie je nach Framework einen passenden Wrapper:

#### Für React:

```tsx
import { useEffect, useRef } from "react";
import { createImage, ImageArgs } from "./Image";

const Image = (props: ImageArgs) => {
  const containerRef = useRef<HTMLDivElement>(null);

  useEffect(() => {
    if (containerRef.current) {
      const imageElement = createImage(props);
      containerRef.current.innerHTML = "";
      containerRef.current.appendChild(imageElement);
    }
  }, [props]);

  return <div ref={containerRef} />;
};

export default Image;
```

#### Für Vue:

```tsx
<template>
  <div ref="container"></div>
</template>

<script setup lang="ts">
import { onMounted, ref, watch } from "vue";
import { createImage, ImageArgs } from "./Image";

const props = defineProps<ImageArgs>();
const container = ref<HTMLDivElement>();

const renderImage = () => {
  if (container.value) {
    const imageElement = createImage(props);
    container.value.innerHTML = "";
    container.value.appendChild(imageElement);
  }
};

onMounted(renderImage);
watch(props, renderImage);
</script>
```

### Beispiel-Implementierung

```tsx
// Using the Image component
const MyPage = () => {
  return <Image src="/path/to/image.jpg" ratio="16:9" alt="Description" />;
};
```

## Best Practices

1. **TypeScript**: Verwenden Sie immer die bereitgestellten TypeScript-Typen für eine korrekte Typ-Überprüfung:

```ts
import type { ImageArgs } from "./Image";
```

2. **CSS-Klassen**: Unsere Komponenten verwenden TailwindCSS-Klassen. Stellen Sie sicher, dass Ihr Build-Prozess diese Klassen in Ihrem finalen CSS-Bundle enthält.

3. **Barrierefreiheit**: Halten Sie die WCAG-Konformität ein, indem Sie bei der Erweiterung von Komponenten entsprechende Alt-Texte und ARIA-Labels bereitstellen.

4. **Performance**: Die Komponenten sind standardmäßig leichtgewichtig. Vermeiden Sie beim Wrapping unnötige Re-Renders.
