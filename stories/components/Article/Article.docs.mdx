## Accessibility

The Article component follows WCAG guidelines and best practices to ensure an accessible experience for all users. Here's a concise overview of its accessibility features:

### Semantic Structure

The component uses semantic HTML elements and ARIA attributes to provide clear structure and context for assistive technologies. The `<article>` element properly identifies content as a self-contained composition.

### Keyboard Interaction

Keyboard users can navigate through the component using standard Tab navigation. The article is focusable and responds to Enter key activation, maintaining a logical tab order through all interactive elements.

### Screen Reader Support

The component implements appropriate ARIA labels, roles, and states to ensure screen readers can effectively communicate the content and state changes to users. Images include descriptive alt text, and dynamic content changes are properly announced.
