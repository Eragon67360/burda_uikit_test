## Accessibility

Icons can serve different purposes in an interface and should be implemented with appropriate accessibility considerations:

1. **Decorative Icons**

   - Use `ariaHidden={true}` for purely decorative icons
   - No role or aria-label required

2. **Meaningful Icons**

   - Set `role="img"` and provide `ariaLabel` when the icon conveys meaning
   - Ensure the aria-label clearly describes the icon's purpose

3. **Interactive Icons**

   - Set `focusable={true}` and appropriate `role` (e.g., "button")
   - Provide clear `ariaLabel` describing the action
   - Ensure proper keyboard navigation support

4. **Icons with Text**
   - When accompanied by visible text, use `ariaHidden={true}` to prevent redundant announcements

<Source
  language="html"

</svg>

<!-- Interactive icon -->

</svg>

  </svg>

`} />
