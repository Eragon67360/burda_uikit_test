## Usage

The ButtonCTA component is a versatile call-to-action button that supports multiple variants, states, and configurations. It's designed to provide clear visual hierarchy and interactive feedback while maintaining accessibility.

## Accessibility

The ButtonCTA component follows WCAG 2.1 guidelines and implements the following accessibility features:

- Proper focus management with visible focus indicators
- ARIA labels for buttons with icons
- Disabled state handling
- Keyboard navigation support
- Color contrast compliance
- Interactive state feedback

### Keyboard Navigation

- <kbd>Tab</kbd>: Focuses the button
- <kbd>Enter</kbd> or <kbd>Space</kbd>: Activates the button
- <kbd>Shift + Tab</kbd>: Moves focus to previous element

## Customization

The ButtonCTA component supports several customization options:

### Variants

- `PRIMARY`: Main call-to-action style
- `SECONDARY`: Alternative emphasis style
- `TERTIARY`: Minimal emphasis style
- `LARGE`: Full-width prominent style
- `LARGE_LIGHT`: Light version of large style
- `LARGE_SUBSCRIPTION`: Special subscription CTA style
- `LARGE_CART_PAY`: Cart payment button style
- `LARGE_LOGIN`: Login form button style

### States

- Default: Base button state
- Hover: Visual feedback on mouse over
- Active: Pressed state
- Focus: Keyboard focus indicator
- Disabled: Inactive state

### Icon Placement

- Right-aligned (default)
- Left-aligned (using `iconLeft` prop)

### Nesting

- Standalone (default)
- Nested within other components (using `nested` prop)
