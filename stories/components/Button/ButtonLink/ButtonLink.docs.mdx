## Usage

The ButtonLink component provides a link-styled button that can be used for navigation or action triggers. It supports icons, different states, and custom styling.

## Accessibility

### Best practices

The ButtonLink component follows accessibility best practices:

- Uses semantic `<a>` element for proper navigation
- Supports keyboard navigation and focus states
- Provides proper ARIA attributes for disabled state
- Maintains sufficient color contrast
- Icons are marked as decorative with `aria-hidden="true"`
- Focus indicators are visible and meet WCAG requirements

### Keyboard Navigation

- <kbd>Tab</kbd>: Focuses the button link
- <kbd>Enter</kbd>/<kbd>Space</kbd>: Activates the link
- When disabled, the link is removed from tab order

## Customization

The component supports different:

- Icon positions (Left, Right)
- Icons from the system icon registry
- States (Default, Hover, Active, Focus and Disabled)
